# Lab 3: Build a Humanoid URDF

## Objective
Design and implement a Universal Robot Description Format (URDF) model for a humanoid robot.

## Prerequisites
- ROS 2 installed with rviz2
- Basic understanding of 3D coordinate systems
- Knowledge of robot kinematics concepts

## Steps

### 1. Create URDF package structure
```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_cmake my_humanoid_description
cd my_humanoid_description
mkdir urdf meshes launch
```

### 3. Create a launch file to visualize the robot
Create `launch/display.launch.py`:

```python
from launch import LaunchDescription
from launch.substitutions import Command
from launch_ros.actions import Node
from ament_index_python.packages import get_package_share_directory
import os


def generate_launch_description():
    pkg_share = get_package_share_directory('my_humanoid_description')
    urdf_file = os.path.join(pkg_share, 'urdf', 'humanoid.urdf')

    robot_state_publisher = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        name='robot_state_publisher',
        output='screen',
        parameters=[{
            'robot_description': Command(['xacro ', urdf_file])
        }]
    )

    joint_state_publisher = Node(
        package='joint_state_publisher_gui',
        executable='joint_state_publisher_gui',
        name='joint_state_publisher_gui',
        output='screen'
    )

    rviz2 = Node(
        package='rviz2',
        executable='rviz2',
        name='rviz2',
        output='screen'
    )

    return LaunchDescription([
        robot_state_publisher,
        joint_state_publisher,
        rviz2
    ])
```

### 4. Update package.xml
Edit `package.xml` to add dependencies:

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_humanoid_description</name>
  <version>0.0.0</version>
  <description>URDF description for a simple humanoid robot</description>
  <maintainer email="your.email@example.com">Your Name</maintainer>
  <license>Apache License 2.0</license>

  <buildtool_depend>ament_cmake</buildtool_depend>

  <depend>joint_state_publisher</depend>
  <depend>joint_state_publisher_gui</depend>
  <depend>robot_state_publisher</depend>
  <depend>rviz2</depend>
  <depend>xacro</depend>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

### 5. Build and visualize the robot
```bash
cd ~/ros2_ws
colcon build --packages-select my_humanoid_description
source install/setup.bash
ros2 launch my_humanoid_description display.launch.py
```

## Expected Output
RViz2 should open and display the 3D model of your humanoid robot. You should be able to manipulate the joint sliders in the joint_state_publisher_gui to move the robot's joints.

## Troubleshooting
- If the URDF doesn't load, check for syntax errors in the XML
- Ensure all dependencies are installed: `sudo apt install ros-humble-joint-state-publisher-gui ros-humble-robot-state-publisher`
- Check that joint limits are properly defined to avoid singularities
